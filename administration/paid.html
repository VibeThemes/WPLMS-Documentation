Steps to fix inconsitencies issues

Inconsistent issues are those issues which appear for certain usecases only. You can apply below steps to rule out if the issue you are facing is an inconistent issue as it does not have any impact on your site.

Here are a list (not exhaustive) of inconsistent issues which may appear and does not have a proper explanation of why they appear.

  • For administrator user it works and other users it does not work.
  • BuddyPress components stop working, groups does not create, messages, notifications stop working.
  • PWA pages turn white or crash.
  • Clicking on a button does not show anything happening.
  • Opening site home page crashes.
  • Site pages start showing 404 error.

There is an explanation for why these inconsistent issues appear and that is most definitely "Cache". A cache is a buffer which records and stores actions that happen in your site to improve the performance. But sometimes it starts conflicting becuase you need a fresh action to occur but the cache keeps showing you a stored action. So the below steps help in clearing these caches.

  1. Permalinks To clear this cache, go to WP admin - Settings - Permalinks and Save the permalinks. MAke sure the permalinks is set to %postname%. Typical use cases : Site pages stop opening, all pages open to 404 or home page.
  2. BuddyPress Pages Go to WP admin - Settings - BuddyPress - Pages and make sure the registrered components have pages associated with them. Typical use cases : Groups not creating, Message replies not sending, emails not working.
